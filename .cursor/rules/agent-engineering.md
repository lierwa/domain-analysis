# Agent 工程约束（同步自 .cursor/rules/agent-engineering.mdc）

在本仓库编写或修改代码时遵守以下约束。

## 开源与调研

- 功能优先调研成熟开源实现；校验、ORM、鉴权、错误处理须用成熟库。
- 自研须有明确理由；设计依据须来自开源项目、官方文档或行业实践。
- 禁止：未调研直接实现、无依据自研基础设施、重复造轮子。

## 设计与实现顺序

- 编码前确认成熟方案、标准实现方式、可参考开源项目。
- 禁止：凭经验设计架构、未确认方案就写代码。

## 抽象与复用

- 重复逻辑出现 ≥2 次须抽象，≥3 次须提取公共模块；优先级：middleware > service > util。
- 禁止：复制粘贴同一能力、重复实现同一模块。

## 代码简洁

- 函数单一职责；控制流嵌套 ≤3；优先可读性而非技巧。
- 禁止：单函数超过约 100 行、深层嵌套。

## 中文注释（核心逻辑）

- 须用中文说明 **WHY** 与 **TRADE-OFF**。
- 禁止：只复述代码在做什么。

```typescript
// ❌ 若 x 为真则返回（表面复述）
// ✅ 提前返回：减少嵌套层级，错误路径更直观（WHY）；可能增加多处 return（TRADE-OFF）
```

## 持续重构与规模

- 每次改动尽量顺带优化旧结构，不增加无谓复杂度；单文件建议 ≤500 行。
- 禁止：只堆新逻辑、职责混乱的模块。

## 输出前自检（不满足则停）

须满足：采用成熟方案、无不妥重复、必要处已抽象、关键逻辑有中文 WHY 注释、有设计依据、复杂度可控。
任一项不满足时停止盲写，先调研、设计或重构。
